# Firebase セットアップ手順書（Hibi 日々 用）

バックグラウンド通知（アプリを閉じていても時間になったら通知が届く）を実現するための Firebase プロジェクト作成ガイド。

所要時間：**15〜20分**。途中でクレジットカード登録が必要になります（無料枠内なら請求 0 円）。

---

## 🎯 このガイドでやること

1. Google アカウントで Firebase プロジェクトを作る
2. Web アプリを登録して、接続情報（config）を取得
3. Cloud Messaging（通知の送信機能）を有効化 & VAPID 鍵を取得
4. Firestore（タスクを保存するデータベース）を作る
5. プランを Blaze（従量課金）に変更 → 無料枠内で 0 円運用
6. 取得した情報を私に共有

最後に **6つの値** を私に渡してもらえれば、コード実装に入れます。

---

## ステップ 1：Firebase プロジェクトを作る

### 1-1. ログイン

ブラウザで以下を開いて、Google アカウントでログイン：
https://console.firebase.google.com/

### 1-2. プロジェクト作成

1. 画面中央の **「プロジェクトを追加」** をクリック
2. プロジェクト名を入力：
   - おすすめ：`hibi-todo`（好きな名前で OK）
   - プロジェクトID は自動生成される（後で使うのでメモ推奨）
3. 「続行」
4. Google アナリティクスの画面が出たら：
   - **「このプロジェクトで Google アナリティクスを有効にする」をオフ**（シンプルにしたいため）
   - 「プロジェクトを作成」
5. 1分くらい待つ → 「新しいプロジェクトの準備ができました」→ **「続行」**

✅ プロジェクトのダッシュボード画面になれば成功

---

## ステップ 2：Web アプリを登録する

### 2-1. アプリ追加

ダッシュボードの中央に「アプリに Firebase を追加して利用を開始しましょう」というエリアがあります。

1. **`</>` （Web）アイコン** をクリック
2. アプリのニックネーム：`Hibi Web`（何でも良い）
3. **「このアプリの Firebase Hosting も設定する」はオフ**（GitHub Pages を使うため）
4. **「アプリを登録」** クリック

### 2-2. ⚠️ 重要：config をコピー

次の画面に、こんなコードが出てきます：

```javascript
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "hibi-todo.firebaseapp.com",
  projectId: "hibi-todo",
  storageBucket: "hibi-todo.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef1234567890"
};
```

**この `firebaseConfig` の中身をまるごとコピーして、あとで私に渡してください。**

5. 「コンソールに進む」をクリック

💡 もし後でコピーし忘れた場合は：
プロジェクト設定（歯車アイコン）→ 「マイアプリ」→ Hibi Web → 「構成」で再表示できます。

## ステップ 3：Cloud Messaging を有効化

通知送信の機能を有効にします。

### 3-1. Cloud Messaging ページへ

1. 左サイドバーの **歯車アイコン（プロジェクト設定）** をクリック
2. 上のタブから **「Cloud Messaging」** をクリック

### 3-2. VAPID 鍵ペアを作成

「ウェブプッシュ証明書」セクションまでスクロール：

1. **「鍵ペアを生成」** ボタンをクリック
2. 生成された文字列（`BK...` で始まる長い文字列）が表示される

✅ **この鍵ペア（Key pair）の値を全部コピーして、あとで私に渡してください。**

💡 後で確認したい場合も、同じページに戻れば表示されます。

---

## ステップ 4：Firestore データベースを作る

タスクを保存するデータベース。

### 4-1. Firestore へ

1. 左サイドバー **「構築」** → **「Firestore Database」** をクリック
2. **「データベースを作成」** をクリック

### 4-2. 設定

1. **ロケーション**：`asia-northeast1`（東京）を選択 → 「次へ」
2. **セキュリティルール**：**「テストモードで開始する」** を選択
   - ⚠️ 30日後に自動で全拒否になります。あとで本番ルールに切り替えるので OK
3. **「有効にする」**

✅ 空のデータベース画面が出れば成功

---

## ステップ 5：Blaze プランに切り替え（クレカ登録）

Cloud Functions を使うために必須。**無料枠内なら請求 0 円** です。

### 5-1. プラン変更

1. 左下の **「Spark プラン」** と書かれた箇所をクリック（画面下のほう）
2. 「プランをアップグレード」画面で **「Blaze」** を選択
3. 請求先アカウントを選択 or 新規作成
4. クレジットカードを登録
5. **予算アラートを設定（強く推奨）**：
   - 「予算アラートを作成」
   - 金額：**¥500** くらいにしておくと安心
   - 設定すると、もし無料枠を超えそうになったらメールで通知

### 5-2. 無料枠の目安（個人利用なら余裕）

| 項目 | 無料枠 | 個人利用想定 |
|------|--------|-------------|
| Cloud Functions 呼び出し | 200万回/月 | 4万回（毎分×24時間×30日） |
| Cloud Functions 実行時間 | 40万GB秒/月 | 数千GB秒 |
| Firestore 読み書き | 5万/日 | 数百 |
| Cloud Messaging | 無制限 | 〜 |

→ 個人で使う分には、まず請求発生しません。

## ステップ 6：Firebase CLI のインストール（任意）

コードをデプロイするために必要。既に入っていればスキップ。

### Mac / Linux

```bash
curl -sL https://firebase.tools | bash
```

### Windows

https://firebase.google.com/docs/cli#windows からインストーラーをダウンロード

### 確認

```bash
firebase --version
```

バージョン番号が出れば OK。

### ログイン

```bash
firebase login
```

ブラウザで Google アカウントにログインを求められる → 許可。

## 📋 次のステップ（私の作業）

上記の情報をもらったら、以下を実装します：

1. **アプリ側のコード**
   - Firebase SDK を読み込み
   - 通知許可を取ってデバイストークン取得
   - タスクを Firestore に同期
   - Service Worker に Firebase Messaging ハンドラ追加

2. **Cloud Function（サーバー側）**
   - 1分ごとに Firestore をチェック
   - リマインド時刻が来たタスクを発見したら FCM で通知送信
   - 送信済みマーキング

3. **デプロイ手順書**
   - `firebase deploy` コマンドで公開
   - Firestore のセキュリティルール更新
